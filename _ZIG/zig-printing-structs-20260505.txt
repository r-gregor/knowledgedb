Printing Zig Structs
April 30th, 2026

   I recently found a new way to print formatted strings in Zig. If you´ve spent much time with the
   language, you´ve probably tried to print something to a console or other output. Here´s an example of
   what that can look like in Zig 0.16.0:

const std = @import("std");

pub fn main(init: std.process.Init) !void {
	const io = init.io;
	const stdout_handle = std.Io.File.stdout();
	var buf: [128]u8 = undefined;
	var stdout = stdout_handle.writer(io, &buf);

	const time = "afternoon";
	const name = "Edward";
	try stdout.interface.print("Good {s}, {s}!\n", .{ time, name });
	try stdout.interface.flush();
}

   Running zig run example.zig produces:
Good afternoon, Edward!

   Similar to printf(3), Zig´s [6]std.Io.Writer.print takes a "format string" that contains both plain
   text and placeholders like {s}, and an list of things to fill those placeholders. Here we´re
   replacing two {s} placeholders with afternoon and Edward respectively.

   The second argument passed is an anonymous struct, but (and now for the point of the post) it can
   also be an existing object. Instead of definging separate variables for time and name, we can wrap
   them up in a Greeting:

const Greeting = struct { name: []const u8, time: []const u8 };
const greeting = Greeting{ .name = "Edward", .time = "afternoon" };

try stdout.interface.print("Good {[time]s}, {[name]s}!\n", greeting);
try stdout.interface.flush();

   We get the exact same output as above.

   What´s nice about this is you don´t need to worry about the order of the fields because they´re
   extracted from the format string. If you squint it looks a little like basic string interpolation.

   Two caveats, though. First is that all fields in the struct must be used in the format string. If I
   add a mood field to Greeting, you have to include it in the format string or you´ll get error: unused
   argument. Second, the struct field must be surrounded by square brackets, like [name], and have a
   format specifier like s or you´ll get error: cannot format slice without a specifier.

   Learning Zig has been great, but it´s quite some distance from the extremely expressive and
   high-level ruby that I´m used to. I´m appreciating any little improvement to ergonomics or
   maintainability like this. And now you know it, too.
