How I emulated class templates in C
2021

Introduction
   I am working on an open source library of tools for C, and the first thing I wanted to create was a
   dynamic Linked List that could accept any data type. While C does require data types to be
   pre-defined, I have managed to create Linked List nodes that accept any data and wanted to share my
   solution.

Void Pointers
   The first step was to use a void pointer to store node data rather than the data itself. A void
   pointer stores a memory address without any type information associated. Using this along with a
   pointer to the next node in the list, my implementation looked like this:

struct Node {
	void *data;
	struct Node *next;
}

Enumerations
   Using void pointers was a good start - I could now store a pointer to any data I wanted and simply
   had to cast it back into its original form when I wanted to use it; however, the user would have to
   allocate their data to the heap prior to node insertion, and I wanted them to simply call an insert()
   function without having to worry about that. Since I could not pass a data type to that function, the
   next best thing was to declare an enumeration of data types:

enum DataTypes {
	Special,
	Int,
	Float,
	Char,
	etc.,
}

Switch Case
   Now users were able to pass in a value from the enumeration to select the desired data type, and
   nodes could be constructed dynamically in the background:

struct Node node_constructor(void *data, int data_type, int size):
	struct Node node;
	switch (data_type) {
		case Int:
		{
			node.data = malloc(sizeof(int) * size);
			for (int x = 0; x < size, x++) {
				((int *)node.data)[x] = ((int *)data)[x];
			}
			break;
		}
		case Float:
			...

   In each case, I allocate the sizeof() for the desired data type multiplied by the parameter size.
   This allows nodes to store arrays as well as single values. Since a single value can be considered an
   array with a single element, the for loop allows for both arrays and single values to be stored
   without knowing which is actually being used. Since the user may want to refer back to the data_type
   and size later, I updated my node struct to accommodate this:

struct Node {
	void *data;
	int data_type;
	int size;
	struct Node *next;
}

Special

   While I included a fair number of data types in my enumeration, users may want to store custom
   structs or a data type added by external libraries. This is where the Special label comes in from the
   DataTypes enumeration. If specified, a user can call malloc manually, instantiate the memory address,
   and simply pass in the pointer. Now the user has the ability to store any kind of conceivable data in
   these Linked List nodes!

Summary
   I am very pleased with this implementation of a Linked List - I made it generic enough so that it can
   be used in any of my future projects and the actual implementation of it in a project is as simple as
   calling a single function. Trying to make a node this dynamic was in a way an attempt to emulate what
   I would have done with a class template in C++, but I learned a lot more by doing it manually in C
   and gained a considerable performance boost in the process!

   Like I said at the beginning, this is part of an open source toolkit I am developing, so you are
   welcome to download the code and use it yourself. You can find it on my GitHub (note that the
   project is almost entirely on the working branch at the time of posting).

   There is a lot more to say about this project, so I will be writing about the Linked List, Queue, and
   other additions to the library as I go here on Reddit. I also like to film my work as I go, so if you
   are into that you may like this developer diary playlist I've been putting together on my
   YouTube.

   Anyway, I hope someone finds all this useful! Thanks for reading!

   Regarding the function pointers, that is exactly what I did! I'm going to do another writeup about
   that soon, but I have two quick points to make here:

   The functions require a pointer to the object - this seemed odd at first, but if you have done any
   Python programming, you will recognize the explicit passing of "self" in class member functions -
   this is the same thing!

   I also wanted to separate the function pointer within the struct from the actual implementation so
   that the user cannot call it directly. To do this, I separated the two into separate files -
   definitions in the header and implementation in the .c file.

   I discuss this at length in the videos I linked!

*** tim36272
   Usually in this case the user provides a comparison function, and you store a pointer to that
   function.

   The function would have a signature and implementation like:

int Less than(const void* lhs, const void* rhs) {
	return (int*)lhs < (int*) rhs);
}

   That way the user can make their "LessThan" function as complicated as they want, including using
   arrays.

*** HomoDeus
