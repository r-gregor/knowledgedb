Understanding Real-Time Video Streaming Basics
Real-time video streaming is the process of continuously transmitting video data from a source (e.g., a
camera) to one or more destinations (e.g., viewers) with minimal delay. Unlike traditional video playback,
where the entire video is downloaded before viewing, real-time streaming sends video data in small chunks,
allowing playback to begin almost immediately.

Key Features of Real-Time Streaming
 * Low Latency: Ensures minimal delay between the video capture and playback.
 * Continuous Transmission: Video data is sent in a steady stream rather than in discrete files.
 * Synchronization: Often synchronized with audio streams to ensure seamless playback.

How Real-Time Video Streaming Works
 * Video Capture: The video source, such as a webcam or video file, captures the raw footage.
 * Encoding: The raw video data is compressed using a codec (e.g., H.264) to reduce size while maintaining
   quality.
 * Packaging: The encoded video is packaged into streamable formats, like MP4 or MKV.
 * Transmission: The video stream is transmitted over a network, often using protocols like RTP
   (Real-Time Protocol) or HTTP Live Streaming (HLS).
 * Decoding and Playback: The client receives the stream, decodes it, and plays it back in near real-time.

Benefits of Real-Time Streaming
 * Interactivity: Enables two-way communication, such as video calls and online gaming.
 * Wide Accessibility: Streams can be accessed on various devices, including smartphones, laptops, and smart
   TVs.
 * Scalability: Modern streaming systems can support thousands or millions of concurrent users.

Working with Video Codecs and Formats
To implement a real-time video streaming program in C, understanding and using appropriate video codecs and
formats is essential. This ensures efficient compression, compatibility, and smooth streaming across devices.

1. What Are Video Codecs?
Video codecs (coder-decoders) are algorithms used to compress and decompress video data. They reduce the
size of video files while maintaining acceptable quality, making real-time streaming feasible. Commonly
used video codecs include:
 * H.264 (AVC): Widely used for its balance of compression efficiency and quality.
 * H.265 (HEVC): Offers higher compression ratios than H.264 but requires more processing power.
 * VP8/VP9: Open-source codecs developed by Google, often used in web streaming.
 * MPEG-4: An older but still relevant codec for compatibility with legacy systems.

2. Understanding Video Formats
Video formats refer to the container files that encapsulate video, audio, and metadata. Common formats
include:
 * MP4: Popular for its support across devices and platforms.
 * MKV: Flexible and capable of holding multiple audio tracks and subtitles.
 * AVI: An older format with widespread compatibility but less efficient compression.
 * FLV: Used for streaming over platforms like Adobe Flash.

3. Choosing the Right Codec and Format
Selecting a codec and format depends on your application requirements:
 * Low Latency Applications: Use H.264 in an MP4 container for a balance between quality and delay.
 * High-Quality Streaming: Opt for H.265 with MKV for better compression without sacrificing quality.
 * Web Streaming: VP8 or VP9 in WebM containers ensures open-source compatibility.

4. Using FFmpeg for Codec Integration
FFmpeg is a powerful library that simplifies working with codecs and formats. To integrate it into your
C program:

Implementing Networking for Video Transmission
Transmitting video data over a network is a core component of real-time video streaming. This step
involves using networking protocols and sockets to send and receive video frames between the server
(stream source) and clients (stream viewers).

1. Networking Basics in C
The C programming language offers low-level socket APIs for network communication. Here´s a brief
overview of essential functions:
 * socket(): Creates a socket.
 * bind(): Associates a socket with a local address and port.
 * listen() and accept(): Used by servers to handle client connections.
 * connect(): Initiates a connection from the client.
 * send() and recv(): Transmits and receives data.

2. Choosing the Right Protocol
Depending on the requirements of your application, you can choose:
 * TCP (Transmission Control Protocol): Reliable, ordered delivery but with higher latency.
 * UDP (User Datagram Protocol): Lower latency but without guaranteed delivery, making it suitable for
 * real-time streaming.
 * RTP (Real-Time Transport Protocol): Built on UDP, designed specifically for streaming multimedia.

3. Creating a Simple Server-Client Model
Server Code Example:

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <arpa/inet.h>
#include <unistd.h>

int main() {
	int server_fd = socket(AF_INET, SOCK_STREAM, 0);
	if (server_fd == -1) {
		perror("Socket creation failed");
		return -1;
	}

	struct sockaddr_in server_addr = {
		.sin_family = AF_INET,
		.sin_port = htons(8080),
		.sin_addr.s_addr = INADDR_ANY
	};

	if (bind(server_fd, (struct sockaddr*)&server_addr, sizeof(server_addr)) == -1) {
		perror("Bind failed");
		close(server_fd);
		return -1;
	}

	listen(server_fd, 1);

	printf("Waiting for client...\n");
	int client_fd = accept(server_fd, NULL, NULL);
	if (client_fd == -1) {
		perror("Accept failed");
		close(server_fd);
		return -1;
	}

	char buffer[1024];
	while (recv(client_fd, buffer, sizeof(buffer), 0) > 0) {
		printf("Received: %s\n", buffer);
	}

	close(client_fd);
	close(server_fd);
	return 0;
}

Client Code Example:

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <arpa/inet.h>
#include <unistd.h>

int main() {
	int client_fd = socket(AF_INET, SOCK_STREAM, 0);
	if (client_fd == -1) {
		perror("Socket creation failed");
		return -1;
	}

	struct sockaddr_in server_addr = {
		.sin_family = AF_INET,
		.sin_port = htons(8080),
		.sin_addr.s_addr = inet_addr("127.0.0.1")
	};

	if (connect(client_fd, (struct sockaddr*)&server_addr, sizeof(server_addr)) == -1) {
		perror("Connect failed");
		close(client_fd);
		return -1;
	}

	char *message = "Hello from client!";
	send(client_fd, message, strlen(message), 0);

	close(client_fd);
	return 0;
}

Optimizing Performance for Real-Time Applications
Real-time video streaming requires high performance and low latency to ensure smooth playback and a
seamless user experience. Optimizing your program involves enhancing both the efficiency of the code
and the system´s ability to handle high-throughput, low-latency video data.

1. Reducing Latency
Key Strategies:
    Lower Buffering: Minimize the size of input/output buffers to reduce processing delays.

setsockopt(socket_fd, SOL_SOCKET, SO_RCVBUF, &buffer_size, sizeof(buffer_size));
setsockopt(socket_fd, SOL_SOCKET, SO_SNDBUF, &buffer_size, sizeof(buffer_size));

    Adjust Frame Rate: Use an appropriate frame rate to balance quality and speed, e.g., 30 FPS for
    standard video.

2. Leveraging Hardware Acceleration
Use GPUs for Encoding/Decoding:
 * GPUs provide hardware-accelerated codecs such as NVENC (NVIDIA) or Quick Sync (Intel), which can handle
    video encoding and decoding faster than CPUs.
 * Example with FFmpeg:

Optimize with SIMD Instructions:
 * Use SIMD (Single Instruction, Multiple Data) operations for pixel processing.
 * Libraries like Intel IPP and AVX can significantly speed up video frame transformations.

3. Multithreading
Parallelize Video Processing Tasks:
    Assign separate threads for video capture, encoding, and transmission.

    Example using pthread:
pthread_create(&capture_thread, NULL, capture_video, NULL);
pthread_create(&encode_thread, NULL, encode_video, NULL);
pthread_create(&transmit_thread, NULL, transmit_video, NULL);

Thread Synchronization:
    Use mutexes or semaphores to synchronize shared resources, such as frame buffers.

4. Optimizing Encoding Settings
Set Compression Parameters Appropriately:
  Reduce computational overhead by adjusting codec settings:
  * Use faster presets for encoding (e.g., FFmpeg´s ultrafast preset).
  * Increase GOP (Group of Pictures) size to reduce keyframe frequency.

codec_ctx->preset = "ultrafast";

5. Efficient Memory Management
