Install Docker in Debian 13 (Trixie) - Step-by-Step Guide
October 14, 2025

   Docker has become one of the most widely used lightweight and portable container systems, running
   consistently across environments. Therefore, if you have just installed the latest Debian 13 (Trixie)
   and want to install Docker, then here are multiple installation methods, from the official repository
   approach to alternatives like rootless mode.

   ---------------------------------------------------------
   Component             Details
   ---------------------------------------------------------
   Docker Version        Docker CE (Community Edition) 24.0+
   Debian Version        Debian 13 (Trixie)
   Installation Time     10-15 minutes
   Difficulty Level      Beginner to Intermediate
   Required Privileges   sudo/root access
   Disk Space Needed     Minimum 2GB free
   ---------------------------------------------------------

Quick Installation
   Those who don't want the explanation of each step can quickly install Docker on Debian 13 Linux using
   the following commands. Others can follow the entire article and video tutorial for a better
   understanding.
# 1. Clean old installations
$> sudo apt remove docker docker-engine docker.io containerd runc

# 2. Install prerequisites
$> sudo apt install -y ca-certificates curl gnupg lsb-release

# 3. Add Docker GPG key
$> sudo install -m 0755 -d /etc/apt/keyrings
$> curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 4. Add repository
$> echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
         https://download.docker.com/linux/debian trixie stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 5. Install Docker
$> sudo apt update && sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin \
   docker-compose-plugin

# 6. Add user to group
$> sudo usermod -aG docker $USER && newgrp docker

# 7. Verify
$> docker run hello-world

What is Docker & Why Debian 13
   Docker is a containerization platform that packages applications with dependencies into portable
   containers. These containers share the host kernel but run isolated, making them more efficient than
   virtual machines.

   Debian 13 (Trixie), the current stable LTS branch, offers a cutting-edge Linux environment with
   modern kernel features complementing Docker's capabilities. This combination provides stability for
   development while maintaining access to newer packages.

Prerequisites & System Requirements

   Before installing Docker, ensure your system meets these requirements:
# Check Debian version
$> cat /etc/debian_version

# Verify architecture (Docker supports x86_64, armhf, arm64, s390x, ppc64le)
$> dpkg --print-architecture

# Check kernel version (minimum 3.10, recommended 5.10+)
$> uname -r

   Your system should have:
     * 64-bit Debian 13 (Trixie) installation
     * Kernel version 5.10 or higher
     * At least 2GB of available disk space
     * An active internet connection for downloading packages
     * User account with sudo privileges

   Update your system packages to ensure compatibility:
$> sudo apt update && sudo apt upgrade -y

   This command refreshes the package index and upgrades existing packages to their latest versions,
   preventing potential conflicts during Docker installation.

Clean Up Old Docker / Conflicts
   If you've previously installed Docker or have conflicting packages, remove them first to ensure a
   clean installation:

# Remove unofficial Docker packages and old versions
$> sudo apt remove docker docker-engine docker.io containerd runc

# Remove Docker directories if they exist
$> sudo rm -rf /var/lib/docker
$> sudo rm -rf /var/lib/containerd

# Clean up any remaining configuration
$> sudo apt autoremove -y
$> sudo apt autoclean

   These commands remove any existing Docker installations, including unofficial packages from Debian
   repositories. The docker.io package from Debian repos often lags behind the official Docker releases,
   so removing it ensures you'll install the latest version.

Install via Docker Official APT Repository on Debian 13 (Recommended)
   The official Docker repository method ensures you receive the latest stable releases and security
   updates directly from Docker Inc. Here's the complete installation process:

Step 1: Install Required Packages
# Install packages for repository management over HTTPS
$> sudo apt install ca-certificates curl gnupg lsb-release

   These packages enable secure HTTPS connections to Docker's repository. The ca-certificates package
   contains certificate authorities, while gnupg handles GPG key verification for package authenticity.

Step 2: Add Docker's Official GPG Key
   Create the keyrings directory if it doesn't exist
$> sudo install -m 0755 -d /etc/apt/keyrings

   Download and add Docker's GPG key
$> curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

   Set proper permissions
$> sudo chmod a+r /etc/apt/keyrings/docker.gpg

   The GPG key verifies that packages actually come from Docker and haven't been tampered with. The
   --dearmor flag converts the ASCII-armored key to binary format for APT's use.

Step 3: Set Up the Repository on Debian 13
   Add official Docker repository to APT sources on Debian 13.
$> echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
         https://download.docker.com/linux/debian trixie stable" | \
         sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

   This command creates a new APT source file specifically for Docker. Note the use of "trixie" in the
   repository URL-this tells APT to fetch packages compatible with Debian 13. The signed-by parameter
   links the repository to the GPG key we added earlier.

Step 4: Install Docker Engine on Debian 13
   Update the package index with Docker packages
$> sudo apt update

   Finally, install Docker Engine, CLI, and containerd
$> sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

   This installs the complete Docker stack: the daemon (docker-ce), command-line tools (docker-ce-cli),
   container runtime (containerd.io), and useful plugins for building images and managing
   multi-container applications.

Alternative Installation Methods

Method 1: Manual .deb Package Installation
   For offline installations or specific version requirements: Visit
   https://download.docker.com/linux/debian/dists/trixie/pool/stable/ and download the .deb files
   for your architecture, then install the Debian packages using:
$> sudo dpkg -i ./containerd.io_*.deb \
  ./docker-ce_*.deb \
  ./docker-ce-cli_*.deb \
  ./docker-buildx-plugin_*.deb \
  ./docker-compose-plugin_*.deb

Method 2: Convenience Script (Development Only)
   Docker provides an automated script for quick installations in development environments:
$> curl -fsSL https://get.docker.com -o get-docker.sh
$> sudo sh get-docker.sh

   Warning: Never use this script in production environments as it doesn't allow version selection and
   may make unexpected system changes.

Method 3: Rootless Mode
   For enhanced security without sudo requirements:

# Install rootless dependencies
$> sudo apt install -y uidmap dbus-user-session

# Install rootless Docker
$> curl -fsSL https://get.docker.com/rootless | sh

# Add to PATH
$> echo 'export PATH=/home/$(whoami)/bin:$PATH' >> ~/.bashrc
$> source ~/.bashrc

Docker Desktop
   For GUI management and additional developer tools, Docker Desktop provides a comprehensive
   solution. Download the .deb package from Docker's website and install it with:
$> sudo apt install ./docker-desktop-*.deb

Post-Installation Steps

Add User to Docker Group
   Avoid using sudo for every Docker command by adding your user to the Docker group:

# Create docker group (if not exists)
$> sudo groupadd docker

# Add cu rent user to docker group
$> sudo usermod -aG docker $USER

# Apply group changes
$> newgrp docker

   Security Note: Adding users to the Docker group grants root-equivalent privileges. Only add trusted
   users who require Docker access.

Configure Docker Daemon (optional)
   Optimize Docker's behavior with a custom daemon configuration:

# Create daemon configuration
$> sudo tee /etc/docker/daemon.json <<EOF
{
	"log-driver": "json-file",
	"log-opts": {
		"max-size": "10m",
		"max-file": "3"
	},
	"storage-driver": "overlay2"
}
EOF

# Restart Docker to apply changes
$> sudo systemctl restart docker

Verify Installation
   Confirm Docker is working correctly:

# Check Docker version
$> docker --version

# Verify daemon is running
$> sudo systemctl status docker

# Run test container
$> docker run hello-world

How to Uninstall / Rollback
   If you need to remove Docker completely:

# Stop all containers
$> docker container stop $(docker container ls -aq)

# Remove Docker packages
$> sudo apt purge -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Remove Docker data and configurations
$> sudo rm -rf /var/lib/docker
$> sudo rm -rf /etc/docker
$> sudo rm /etc/apt/sources.list.d/docker.list
$> sudo rm /etc/apt/keyrings/docker.gpg

# Remove group
$> sudo groupdel docker

FAQ
   Q1: Can I install Docker using Debian's default repositories?
   A: While Debian includes a docker.io package, it's often outdated. The official Docker repository
   provides the latest stable version with timely security updates.

   Q2: Is Docker compatible with Debian 13?
   A: Yes, Docker fully supports Debian stable and testing branches. However, be prepared for occasional
   package dependency changes as Debian 13 evolves.

   Q3: How do I switch between Docker versions?
   A: Use apt's version selection: sudo apt install docker-ce=<VERSION> after listing available versions
   with apt-cache madison docker-ce.

   Q4: How much disk space does Docker typically use?
   A: Base installation requires about 500MB, but container images and volumes can quickly consume
   several gigabytes. Monitor usage with docker system df.
