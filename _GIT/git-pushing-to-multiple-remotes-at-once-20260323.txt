Pushing to multiple git remotes at once
2026-03-20

   I recently learned that it was possible to configure git to push to several remotes at once. I wanted
   to do this for this very blog, where I somehow rely on 2 distinct git repositories:
     * a simple bare repo simply located somewhere on my server, that triggers the blog deployment
       through a post-receive hook.
     * a forgejo repo that is used for public archiving

   I didn´t want to deploy the site via forgejo, because it´s running in its own container, and the blog
   is served directly on the server, so that would probably have been more complicated than necessary to
   setup. What it meant though, was that I always had to run the following commands after having made a
   change:
$> git push origin master
$> git push forgejo master

   I recently learned that you can configured multiple URLs when pushing to a remote, which means that
   git push origin master would push to all remotes.

   Assuming that you want to configure the origin remote, you can run

# Add the current pull URL as a push URL
$> git remote set-url --add --push origin $(git remote show origin | grep "Fetch URL" | awk '{ print $3 }')

# Add a new URL as another push URLs
$> git remote set-url --add --push <new url>

# You can then inspect the push URLs
$> git remote show origin | grep "Push"

   That´s all there is to it!
