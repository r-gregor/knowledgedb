***
Everyone seems to be taking the "here´s a fish" approach, rather than a "here´s how to fish" approach. So,
here´s some explanation of what´s going on and how to get your windows in that configuration:

i3wm uses a tree structure to represent the window structure. Here´s a general idea of the tree at the start:

      horiz
 _______|______
 |      |      |
win1   win2   win3

where horiz is the default layout of the entire screen, and child relationships indicate that they are
part of that layout. Once you follow these steps:

Focus Window 2.
$mod+v (or whatever your shortcut is for vertical mode)

Your tree structure will look like this:

      horiz
 _______|______
 |      |      |
win1   vert   win3
        |
       win2

where vert is a vertical container containing a single window (in this case win2).

Then, you want to move win3 into the vert, so you can do so using $mod+shift+left and your tree will
now look like this:

  horiz
 ___|____
 |      |
win1   vert
    ____|____
    |       |
   win2   win3

If you always keep this implicit tree structure in mind, you will never have any issues getting your
windows to the right places on your screen.

If all three windows are already opened and you just want to move them first focus window 3 and move it
up, then focus window 1 and move it left.

Or you could first focus window 3, split vertically (mod+v), then focus window 2 and move it right.

***
Assuming you started with three windows already opened:
  * Use mod+j and mod+semicolon to change focus to window 2.
  * Use mod+Shift+l to move focused window (window 2) up. It´s going to take up the whole top half of
    your screen.
  * Use mod+k, mod+j and mod+semicolon to change focus to window 1.
  * mod+Shift+j to move window 1 left.

The logic here is following: If there is only one way for window to move (in this example up), then
the window will occupy the whole upper half of the screen. Same holds for window one. There is only one
way it can move more left (it´s already on bottom left) so it stretches over the whole left half of the
screen and the other two split the right half, the top one staying up the the bottom one down.

I cannot really explain it better than this but the official documentation can.

I looked at what I do and a common operation is to arrange two windows that are side-by-side as
one-above-the-other and vice versa. I do this:

    # Toggle a side-by-side pair to one-above-the-other.
    # A for Above.
    bindsym a $default, focus right, split v, focus left, move right, move up ;
    # Toggle a one-above-the-other pair to side-by-side.
    # S for Side by Side.
    bindsym s $default, focus down, split h, focus up, move down, move left;

OVERVIEW
i3tree is a program I wrote to help myself adjust to version 4 of the i3 tiling window manager. It displays
the containers your windows are children of and what layout those containers are using.

INSTALLATION
i3tree requires an installation of the i3 window manager, perl, and the following perl module: AnyEvent::I3

USAGE
Just run it.

EXAMPLES

$> i3tree
Workspace 1 (horizontal)
        Window foo@bar: ~
        Split (vertical)
                Window foo@bar: ~
                Window foo@bar: ~
Workspace 2 (vertical)
        Window foo@bar: ~
        Window foo@bar: ~

CONVENIENCE
You could either start i3tree from a running terminal, or launch it inside a floating window. Here´s the
configuration needed:

for_window [title="^i3tree$"] floating enable
bindsym $mod+t exec xterm -hold -title i3tree -e ~/path/to/i3tree

LIMITATIONS
i3tree shows containers from all workspaces. It could be useful to optionally limit this to only the current
workspace.
